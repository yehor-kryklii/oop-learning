# RECIPE

### Описание программы

```

Cоздать класс эмулирующий
кулинарный рецепт (Recipe). Каждый рецепт подразумевает 
некоторое количество различных нгредиентов, а также 
шагов по приготовлению

```

### Описание операции

```

--Конструктор, принимающий название рецепта, описание
рецепта и имя автора рецепта в виде строк.
	
	Ожидаемые сообщения:
	
		-Messages::EmptyRecipeName - если передано
			пустое название рецепта.
		-Messages::EmptyRecipeDescription - если
			передано пустое описание рецепта.
		-Messages::EmptyRecipeAuthor - если передано
			пустое имя автора.

--Метод getName - возвращает название рецепта

--Метод getDescription - возвращает описание рецепта

--Метод getAuthor - возвращает имя автора рецепта

--Метод getIngredientsCount - возвращает количество
ингредиентов, предусмотренных рецептом

--Пара методов beginIngredients / endIngredients,
возвращающих пару итераторов по ингредиентам
рецепта. Итератор должен указывать на пару, первым
элементом которой является название ингредиента в виде
строки, а вторым - вес данного ингредиента в виде целого
числа гр.

--Метод hasIngredient - принимает название ингредиента в
виде строки и возвращает true, если такой ингредиент
представлен в рецепте

--Метод addIngredient - принимает название ингредиента в
виде строки и вес ингредиента в виде положительного
целого числа в граммах - и добавляет его к рецепту.

	Ожидаемые сообщения:

		-Messages::EmptyIngredientName - если
			передано пустое название ингредиента.
		-Messages::IngredientValueMustBePositive - если
			переданный вес инргединета не является
		положительным числом.
		-Messages::DuplicateIngredient - если данный
			ингредиент уже присутствует в рецепте.

--Метод modifyIngredient - принимает название ингредиента
в виде строки и вес ингредиента в виде положительного
целого числа в граммах - и обновляет вес ранее
добавленного ингредиента

	Ожидаемые сообщения:

		-Messages::EmptyIngredientName - если
			передано пустое название ингредиента.
		-Messages::IngredientValueMustBePositive - если
			переданный вес инргединета не является
			положительным числом.
		-Messages::IngredientCannotBeFound - если
			такой ингредиент не был зарегистрирован
			ранее.

--Метод getIngredientValue - принимает название
ингредиента в виде строки - и возвращает его вес в виде
целого числа в граммах

	Ожидаемые сообщения:

		-Messages::EmptyIngredientName - если
			передано пустое название ингредиента.
		-Messages::IngredientCannotBeFound - если
			такой ингредиент не был зарегистрирован
			ранее.

--Метод getCookStepsCount - возвращает количество
добавленных шагов приготовления

--Метод getCookStep - возвращает шаг приготовления с
указанным порядковым номером в виде строки

	Ожидаемые сообщения:

		-Messages::IndexOutOfRange - если порядковый
			номер шага некорректен.

--Метод addCookStep - принимает действие по
приготовлению в виде строки, и добавляет к рецепту
	
	Ожидаемые сообщения:
	
		-Messages::EmptyCookStep - если передано
			пустое название действия по приготовлению.

--Перегруженные операторы == и !=, которые учитывают все
100% данных рецепта (название, описание, имя автора,
состав ингредиентов с весами и список шагов
приготовления)

--Также, класс должен запрещать копирование и присвоение объектов

```
